Производительность

#2 — Алгоритмическая сложность O(n²)

Оператор x not in unique_lst выполняет линейный поиск по списку — это O(n) на каждой итерации, итого O(n²) в целом. На 14 элементах незаметно, но на 1 000 000 элементах разница катастрофическая.

Проверка принадлежности через set — это O(1) в среднем, что даёт общую сложность O(n).

# O(n) — с сохранением порядка
seen = set()
result = []
for x in lst:
    if x not in seen:
        seen.add(x)
        result.append(x)

Либо, начиная с Python 3.7+ (словари сохраняют порядок вставки):

# Однострочник O(n)
result = list(dict.fromkeys(lst))

Эталонное решение для Python 3.14

Если бы нужно было написать «по-взрослому»:

"""Модуль для удаления дубликатов из списка с сохранением порядка."""


def remove_duplicates[T](items: list[T]) -> list[T]:
    """Удаляет дубликаты, сохраняя первое вхождение каждого элемента.

    Args:
        items: Исходный список (элементы должны быть хешируемыми).

    Returns:
        Новый список без дубликатов в исходном порядке.
    """
    seen: set[T] = set()
    result: list[T] = []
    for item in items:
        if item not in seen:
            seen.add(item)
            result.append(item)
    return result


if __name__ == "__main__":
    numbers = [1, 2, 3, 4, 4, 4, 4, 4, 5, 5, 6, 7, 7, 9]
    print(remove_duplicates(numbers))  # [1, 2, 3, 4, 5, 6, 7, 9]
